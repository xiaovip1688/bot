### 1. HTTP 404 接口（72个）

## 关键发现

### 1. pybit SDK 与实际 API 不一致

**问题：** pybit SDK v5.14.0 中定义的许多接口在实际 Bybit V5 API 中返回 404

**影响模块：**
- CryptoLoan-Fixed（15个接口全失败）
- PreUpgrade（6个接口全失败）
- LeverToken（3个接口全失败）
- User 子账户管理（8个接口失败）

**建议：**
- 以实际 API 文档为准
- SDK 仅作参考
- 对关键接口进行验证

### 2. 权限体系复杂

**发现：**
- 单个 API Key 无法测试所有接口
- 需要 7+ 种不同权限
- 部分接口需要特殊账户类型（经纪商）

### 3. 业务逻辑限制

**发现：**
- 部分接口受账户状态限制
- 统一账户无法使用某些功能
- 有持仓时无法切换模式

## 测试建议

### 对于 API 使用者

1. **先验证接口** - 使用前先测试接口是否可用
2. **申请正确权限** - 根据需求申请对应的 API Key 权限
3. **处理错误** - 做好错误处理和重试机制
4. **参考官方文档** - SDK 可能有滞后

### 对于进一步测试

1. **申请权限** - 开通 Wallet/Withdraw/Exchange/Fiat 等权限
2. **确认接口** - 与 Bybit 官方确认 404 接口状态
3. **多账户测试** - 使用不同账户类型测试
4. **自动化** - 建立持续测试机制
